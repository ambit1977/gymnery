# 指示書: インターバルタイマー終了アラート実装（Wake Lock + 音）

## 背景と目的

このアプリはトレーニング記録PWA（ホーム画面追加・`display: standalone`・GitHub Pagesホスト）。
セット間のインターバルタイマー（デフォルト60秒）があるが、終了時のアラートが実質機能していない：

- `navigator.vibrate` を呼んでいるが **iOSは Vibration API 未対応**のため無音・無振動（ガード付きなので静かに何も起きない）
- インターバル中にiPhoneがスリープするとJSが凍結し、タイマー表示も止まる

**今回のゴール**: インターバル中は画面をスリープさせず（Screen Wake Lock API）、終了時に音＋視覚フラッシュで確実に知らせる。サーバーや通知許可は使わない（Web Push化は別案として保留中。今回のスコープ外）。

すべてリポジトリ直下のフラットな vanilla JS 構成。ビルド工程なし。

## 実装内容

### 1. Screen Wake Lock（スリープ防止）

- `startIntervalTimer()` 内で `navigator.wakeLock.request('screen')` を取得
- タイマーがクリアされる全経路（保存フローL995、モーダル/画面の破棄経路）で release
- タイムアップ後もタイマー表示（超過カウントアップ）が続く間は保持したままでよい
- **再取得処理が必須**: ユーザーが手動でロック→復帰したり他アプリに切り替えるとOSがwake lockを自動解放する。`visibilitychange` で `document.visibilityState === 'visible'` かつ `intervalTimerId` が生きていれば再取得する
- フィーチャーディテクション必須（`'wakeLock' in navigator`）。取得失敗（低電力モード等）は握りつぶしてアプリ動作に影響させない
- 既知情報: iOS SafariはiOS 16.4+で対応。ただし**ホーム画面PWAではiOS 18.4未満にwake lockが効かないWebKitバグがある**（Bug 254545）。18.4+なら動く。失敗しても他機能は動くこと

### 2. 終了音（iOSで実際に鳴る方式で）

重要な制約:

- **Web Audio API（oscillator）はiOSのサイレントスイッチでミュートされる**。ジムではマナーモードの可能性が高い
- **`<audio>` 要素によるメディア再生はサイレントスイッチを無視して鳴る**（メディア音量に従う）
- iOSでは音声再生の「アンロック」にユーザージェスチャが必要。`startIntervalTimer` はボタンの click ハンドラなのでここでアンロックできる

実装方針:

1. ビープ音のWAVデータをJSでプログラム生成する（外部音源ファイル追加禁止。オフライン動作を壊さないため）。例: 880Hz・0.2秒のビープを0.15秒間隔で3回、16bit/モノラル/22050Hzで生成し Blob URL 化して `<audio>` 要素にセット。生成は初回のみ、要素は使い回す
2. `startIntervalTimer()`（ジェスチャ内）で `audio.play()` → 即 `pause()` + `currentTime = 0` でアンロック
3. タイムアップ時（`hasTriggeredEnd` を立てる箇所）で `audio.play()`。失敗時は Web Audio oscillator にフォールバック
4. 既存の `navigator.vibrate` 呼び出しはAndroid用にそのまま残す

### 3. 視覚アラート

- タイムアップ時に `#interval-display` またはコンテナに点滅アニメーション用クラスを付与（`style.css` に keyframes 追加、既存の `timer-danger` 系の配色トーンに合わせる）
- 既存の `showToast` があるので「インターバル終了 ⏱」等を表示してよい

### 4. その他

- `addOneMinuteToInterval()` は変更不要（wake lock保持中のため延長はそのまま機能する）
- `startIntervalTimer` の連打（タイマー再スタート）で wake lock やaudioが二重取得・リークしないこと
- `sw.js` の `CACHE_NAME` を `training-app-v33` にバンプ（このリポジトリの更新反映の慣習）

## 受け入れ条件

実機のiPhone（ホーム画面追加済みPWA）で:

1. 「＋ インターバル」タップ後、放置しても60秒間画面がスリープしない（iOS 18.4+）
2. 60秒経過時に**マナーモードでも**音が鳴り、表示が点滅する
3. 「＋1分」延長後も同様に動作する
4. 記録保存・画面を閉じた後は通常どおりスリープする（wake lockが解放されている）
5. Androidでは従来どおりバイブも動く
6. Wake Lock非対応環境（iOS 18.4未満のPWA等）でもエラーにならず、音・点滅は動く

## スコープ外

- Web Push通知（ロック画面通知）: 今回の案が実用にならなかった場合の次案。Cloudflare Worker等のサーバーが必要になるため別タスク
- セッション全体タイマー（`startTimer`, L138付近）のバイブ箇所の音化: 触らない
